Автоматизированные тесты для Яндекс.Самокат

Цель проекта
Автоматизация проверки ключевой функциональности веб-приложения сервиса аренды самокатов с использованием Selenium WebDriver и pytest.

Покрытие тестами
-FAQ (Вопросы о важном) — проверка раскрытия аккордеона и соответствия текста ответов (8 тестов)
-Оформление заказа — полный цикл заказа через две точки входа (хедер и главная страница) (2 теста)
-Логотипы — проверка редиректов (Яндекс → Дзен, Самокат → главная) (2 теста)

Технологический стек
-Язык: Python 3.14.2
-Фреймворк: pytest
-WebDriver: Selenium + Firefox Driver
-Отчётность: Allure
-Паттерн проектирования: Page Object Model (POM)
-Ожидания: Explicit Waits (WebDriverWait)
-Версионирование: Git, GitHub

Статистика
-Всего тестов: 12
-Процент прохождения: 100% (12/12)
-Время выполнения: ~60-110 секунд
-Стабильность: 100% 

Ключевые достижения
-Устранение циклических зависимостей: вынесение URL в отдельный модуль urls.py, данных в data/
-Оптимизация ожиданий: замена time.sleep() на WebDriverWait (повышение стабильности)
-Data-Driven подход: параметризация тестов, вынос данных в отдельные модули
-Переиспользование кода: создание BasePage с общими методами (скролл, клики, ожидания)
-Чистая архитектура: разделение слоёв (данные → Page Object → тесты)
-Профессиональная отчётность: интеграция Allure с детализацией шагов
