Автоматизированные тесты для API сервиса доставки самокатов

Цель проекта
Автоматизация тестирования REST API веб-приложения сервиса доставки самокатов с проверкой основных бизнес-сценариев и валидации ответов сервера

Покрытие тестами
- Courier API — проверка регистрации, авторизации и удаления курьеров (14 тестов)
- Orders API — полный цикл работы с заказами: создание, получение списка, принятие, отмена (26 тестов)
- Валидация — проверка обработки ошибочных запросов и граничных значений (9 тестов с xfail)

Технологический стек
- Язык: Python 3.14.2
- Фреймворк: pytest 7.4.0
- HTTP-клиент: requests 2.31.0
- Отчётность: Allure 2.13.2
- Паттерны: API Client (аналог Page Object для API)
- Фикстуры: pytest fixtures с setup/teardown
- Параметризация: @pytest.mark.parametrize
- Версионирование: Git + GitHub

Статистика
- Всего тестов: 40
- Процент прохождения: 100% (31 passed, 9 xfailed)
- Время выполнения: ~60-75 секунд
- Стабильность: 100% 

Ключевые достижения
- Независимость тестов: каждая фикстура управляет своими данными (создание → тест → очистка)
- Переиспользование кода: фикстуры авторизованного курьера и заказа с курьером
- Data-Driven подход: параметризация тестов на создание заказа с разными цветами
- Обработка багов API: 9 тестов помечены как xfail с описанием известных проблем
- Чистая архитектура: разделение на слои (API clients → models → tests → fixtures → utils)
- Профессиональная отчётность: интеграция Allure с детализацией по шагам (feature/story/step)

Найденные баги API
1.	409 вместо 400 — дубликат логина возвращает Conflict вместо Bad Request
2.	404 вместо 400 — несуществующий пользователь/заказ возвращает Not Found
3.	Зависание /login — запрос без поля password вызывает таймаут
4.	Отсутствие валидации — обязательные поля заказов не проверяются
5.	500 вместо 400 — отсутствие order_id в URL вызывает Internal Server Error
